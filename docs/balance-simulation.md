# バランスシミュレーション

スキル宣言値そのものの設計とDPS比較には、先に決定論的な [`スキルパワー設計式`](skill-power-formula.md) を使う。静的計算は戦闘を実行せず、発動間隔、条件難度、資源倍率、価格、レア度から全スキルの予算比を出す。この文書のシミュレーションは、その式では測れない実盤面の勝率・採用差を確認するときだけ使う。

UIを起動せず、実際の敵生成・回路解析・戦闘処理を使って多数の戦闘を再生し、スキル単位の傾向をJSON・CSV・Markdownへ出力する。

## 基本コマンド

```bash
pnpm balance:simulate
```

既定値では、固定シード `20260721`、run 1〜9、通常戦闘10,000戦、各スキル最大40件の厳密差し替え試験と、各スキルを必ず含む40件の効果無効化試験を実行し、次のファイルを生成する。現在の基準データでは約2分で完了する。

```text
reports/balance/latest.json
reports/balance/latest.csv
reports/balance/latest.md
```

`latest.*` はローカル比較用のためGit管理しない。

主要オプション:

```bash
pnpm balance:simulate -- --battles 10000 --runs 3:9 --seed 20260722
pnpm balance:simulate -- --skills strike,poison-needle --skill-trials 50
pnpm balance:simulate -- --output reports/balance/experiment-a
pnpm balance:simulate -- --help
```

## 計測方法

### 生成ビルド大会

`generateEnemyBuild` で同じrunの盤面を2つ生成し、A対BとB対Aを実行する。ビルドIDは `buildDesign.builds` から動的に選ぶ。両者には同じレベル、HP増加、累積コイン予算を適用する。敵生成は予算を超えず、目標ノード数を買えない場合はノードを減らす。盤面を左右へ入れ替えることで、戦闘処理の陣営差を集計から切り分ける。奇数の戦闘数を指定した場合、最後の1戦だけは入替戦を持たない。

通常大会では次を集計する。

- 全体、run別、ビルドマッチアップ別の勝敗と平均決着tick
- run別のレベル、累積予算、平均使用額、平均ノード数
- スキルの登場戦闘数、勝率、発動回数、各出力
- 同じrun・同じビルドで、そのスキルを持たない盤面と比較した補正スコア率差
- 勝率の95% Wilson区間
- 同レアリティ内の報告出力/発動のZスコア

勝ちは1、引分は0.5、負けは0としてスコア率を計算する。単純勝率とは別の値である。

### 反実仮想ベンチマーク

対象スキルを、次の条件をすべて満たす生成済みノードへ差し替える。

- 同じレアリティ
- ビルド上の役割が1つ以上重なる
- 決め手同士を比べる場合は同じ決め手経路に属する
- 回転後の接続ポート集合が完全に同じ
- 差し替え後も生成盤面の全ノードが通電する

元盤面と差し替え盤面を同じ相手へぶつけ、さらに両者を左右入替して、スコア率、報告ダメージ、防御出力、決着tickの差を取る。直接イベントを出さないチャージ・増幅・加速ノードも、この勝敗差で評価できる。

互換ノードが見つからないスキルは、指定件数より反実仮想サンプルが少なくなる。件数が `minimumCounterfactualSamples` 未満なら外れ値判定には使わない。

### 効果アブレーション

厳密な差し替え候補が存在しない固有形状のスキルも評価するため、対象スキルとリンクしたビルドを選び、そのスキルを必ず含む盤面を生成する。接続・配置・相手を変えずに効果だけ無効化して再戦し、相手側には対象スキルを含まない盤面を使う。レアリティの排出率に依存せず、全playableスキルで指定件数を確保でき、直接traceを出さない補助スキルにも対応する。

無効化差は「そのスキルを別の購入候補へ替えた差」ではなく、「同じ接続を無料で維持したまま効果を失った差」である。同レアリティ内の無効化差Zスコアと、通常大会の補正スコア率を組み合わせて読む。

## 外れ値判定

既定の判定値は次の通り。

- 通常大会で採用・不採用がそれぞれ40登場以上で、少ない側が多い側の10%以上
- 同run・同ビルドの補正スコア率差が±8ポイント以上
- 厳密差し替え試験が24件以上、差が±8ポイント以上、95%区間が0をまたがない
- 同レアリティの報告出力/発動から±2標準偏差以上
- 同レアリティの効果無効化差から±2標準偏差以上
- 同レアリティの効果無効化差が中央値から±16ポイント以上離れ、95%区間も中央値より同じ側

厳密差し替えの条件を満たすか、補正スコア率と同じ向きに出力Zスコア、効果無効化Zスコア、同レアリティ差が外れたスキルを `suspectedOutlier` とする。同レアリティ差が特に大きい場合も要確認へ上げる。強い出力なのに勝率差が負、といった逆向きのシグナルは単独シグナルとして残すが、自動外れ値にはしない。これは自動弱体化の指示ではなく、再現盤面を確認する優先順位である。

`loop`、`fully-connected`、`straight-line` は、固定レイアウトがスキル固有の条件を常に満たすとは限らない。このため同レア差は数値として残すが、配置条件付きスキルの自動外れ値判定には使わない。配置列を確認し、条件を満たす手書き回路テストで上限と実効値を検証する。`free` のスキルだけが同レア差による自動判定の対象になる。

## 新しいスキル・ビルドの自動取り込み

### スキル

- `buildDesign.skills` で `status: "playable"` と有効な `blockId` を持つスキルは、通常大会の集計と効果無効化試験へ自動で入る。
- `status: "planned"` は実装前なので対象外。
- `--skills` を明示した実験だけは指定スキルに限定する。通常の `balance:baseline` と `balance:check` は全playableスキルを再取得する。
- `--skills` で絞ったレポートの同レア差は、指定スキルの中でサンプルを持つ同レアだけが比較母集団になる。集中試験では平均と95%区間を読み、同レア差の最終判定は全スキル版で行う。
- 旧ベースラインの `skillIds` は引き継がない。追加・削除があれば比較条件を不一致にして `balance:check` を失敗させるため、追加スキルが無言で抜けることはない。

### ビルド

- `buildDesign.builds` の追加ビルドは敵生成・通常大会・ビルド間集計へ自動で入る。コード側のビルドID列挙は持たない。
- ビルドIDは宣言した `axisId` の値として存在し、playableな起点と決め手を含む必要がある。各スキルの `buildLinks` と対象軸の値も追加する。
- 予算内で2ノード以上の通電盤面を生成できないビルドはエラーになる。自動的に別ビルドへ読み替えない。
- 追加・削除されたビルドIDも比較条件へ保存し、古いベースラインとの比較を不一致にする。

## ベースライン運用

現在値を基準として保存する。

```bash
pnpm balance:baseline
```

ゲームデータ変更後、同じシード・同じ条件で比較する。

```bash
pnpm balance:check
```

比較条件、playableスキル一覧、ビルド一覧が異なるか、新しい `suspectedOutlier` が発生すると終了コード1になる。既存の外れ値はベースラインに含まれるため、それだけでは失敗しない。

おすすめの調整ループ:

1. `pnpm balance:check` で固定シードの差分を見る。
2. `report.md` で勝率差と出力を確認する。
3. 対象スキルだけ試行数を増やして再計測する。
4. 別シードを検証用として実行し、同じ傾向が出るか確認する。
5. 最終判断後にだけベースラインを更新する。

検証例:

```bash
pnpm balance:simulate -- --battles 10000 --seed 20260722 --output reports/balance/validation
```

調整中は基準シードを変更しない。検証シードの結果を見てから調整値を戻って変更すると、検証データへの過剰適合になるため避ける。

## 数値の制約

- 現在の生成ビルドは敵生成器の固定レイアウトを使い、そのrunでプレイヤーが得ている平均累積コイン以内で構築する。レベル別の排出率も使うが、個々のショップ候補、ロック、リロール、融合、プレイヤーの配置判断は再現しない。
- スキル別ダメージは戦闘traceへ記録された値で、シールド吸収やオーバーキル控除前である。
- 回復は最大HPで制限された実回復量である。
- 毒tickはチーム単位で記録され、毒を付与した個別スキルへ帰属しない。
- ダメージ、毒付与、シールド、回復の単純合計は単位が異なる。出力Zスコアだけで強弱を決めず、補正勝率差と反実仮想差を優先する。
- 自動生成は平均的な盤面を測るもので、最強配置や特定の2〜3枚コンボを探索する最適化ボットではない。蓄積値へ倍率を掛ける決め手、発動間隔短縮、合流倍率などは、想定上限の手書き回路テストを必須とする。
- 採用盤面がほぼ100%または0%のスキルは、不採用または採用の比較母数が不足する。補正勝率差は両方が最低件数を満たし、少ない側が多い側の10%以上ある場合だけシグナルに使う。
- 毒のように「今使うと瞬間火力が出るが、残せば継続ダメージになる」効果は、無効化差だけで価値を断定しない。平均、同レア差、手書きコンボ試験を併読する。

実ゲームの敵は平均値ではなく、そのプレイで実際に獲得した累積コインを上限にする。ショップ選択まで含むラン全体の強さを測る場合は、購入・リロール・融合・再配置を選ぶ戦略ボットが別途必要になる。現段階では敵とシミュレーターの予算・レベル・HP規則を共通化し、購入戦略そのものは対象外とする。
